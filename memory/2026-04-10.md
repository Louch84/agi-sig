### Issues Found:

**BUG-1 [HIGH] trace_logger.py line ~60 — DATA LOSS on trace logging**
- Problem: Was using `open(tmp, "w")` + `os.replace()` on a JSONL file (one object per line). This overwrites the ENTIRE file every single trace, destroying all prior traces.
- Severity: HIGH — silently loses all historical learning data
- Fix: Changed to append mode `open(TRACE_FILE, "a")` ✅ APPLIED

**BUG-2 [MEDIUM] ollama-dispatcher.py — model not loaded before use**
- Problem: `process_task()` sends requests to models that may not be loaded in Ollama, causing timeout failures. No pre-load or health check.
- Severity: MEDIUM — tasks fail or timeout unnecessarily
- Fix: Added `ensure_model()` with /api/tags check + load via generate call ✅ APPLIED

**BUG-3 [MEDIUM] ollama_mem.py — cosine_sim crashes on length mismatch**
- Problem: `zip(a, b)` silently drops elements when vectors have different dimensions. Could cause search results to be wrong or crash.
- Severity: MEDIUM — wrong search scores or crash on malformed embeddings
- Fix: Added length check before zip ✅ APPLIED

**BUG-4 [LOW] ollama-daemon.py — bare `except: pass` hides failures**
- Problem: Multiple bare except:pass blocks swallow errors silently
- Severity: LOW — makes debugging impossible
- Note: Existing pattern across codebase; marked for refactor

**BUG-5 [LOW] scanner/notify_discord.py — placeholder webhook constant**
- Problem: `DISCORD_WEBHOOK_URL = "YOUR_DISCORD_WEBHOOK_URL_HERE"` is a placeholder left in code
- Severity: LOW — would post to wrong place if env var misconfigured

**BUG-6 [INFO] task-planner.py — JSON parsing fragile**
- Problem: Planner relies on finding "{" and "}" lines to extract JSON from LLM output. LLMs frequently add markdown fences or extra text.
- Severity: INFO — planner silently returns None on any parse failure

**BUG-7 [INFO] ollama-daemon.py — `trace_logger.py` path collision**
- Problem: Both scripts/ollama-daemon.py and agi-sig/scripts/ollama_mem.py exist with identical names. The TOOLS.md references scripts/ollama_mem.py but there are two copies.
- Severity: INFO — confusing which one is active
