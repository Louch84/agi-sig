**Problem:** Lou sees black screen when opening `http://localhost:5180/` (React Three Fiber app with Vite dev server)

**What I tried (in order):**
1. Restarted Vite dev server — server was on wrong port (bound to IP instead of port)
2. Checked devtools console — Lou couldn't find it / nothing showing
3. Added `ErrorBoundary` component — didn't help
4. Stripped `PostFX` (Bloom, Vignette) — didn't help
5. Added simple test box (red/yellow/blue) — didn't show
6. Background color renders (WebGL context initializes) but nothing 3D appears
7. Switched to pure vanilla Three.js via CDN (no React) — CDN hanging for Lou's browser
8. Tried Three.js UMD (global script) — **r160 removed UMD support entirely** (Three.js confirmed: `The page at http://localhost:5180/ ran network resources with an unsupported feature (transformers)` — meaning importmaps, which r160 uses exclusively)
9. Wrote local OrbitControls from scratch (avoiding dependency issues)
10. Built production Vite bundle and served on port 5190 — cleanest test, should work

**Root cause hypothesis:** Either:
- GPU/drivers on Lou's MacBook Air don't support WebGL2 features needed by Three.js r160 + post-processing
- Safari/WebKit handles the WebGL context differently
- Lou needs to try `http://localhost:5190/` (bundled production build) in incognito Chrome

**Key lesson:** Three.js r160 (2024) went all-in on ES modules/importmaps and **removed UMD support**. No more `<script src="three.min.js">`. Local builds now require either:
1. npm + bundler (Vite works great)
2. Self-hosted ES module files
